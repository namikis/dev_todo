# Miro → TODO タスク追加スキル

Miroボードの情報を取得し、タスク・サブタスクに分解してTODOリストに登録する。

## ワークフロー

### Step 1: MiroボードURLの取得

以下の優先順位でURLを取得する：

1. ユーザーが引数またはメッセージでURLを指定した場合 → そのURLを使用
2. 指定がない場合 → メモリ (`memory/reference_miro_board.md`) を確認
3. メモリにもない場合 → ユーザーに確認する

URLは `https://miro.com/app/board/xxxxx=/` 形式である必要がある。welcomeonboard形式の場合はブラウザで開いた後のURLを聞き直す。

### Step 2: ボード情報の取得

以下の2つを並列で実行し、ボード全体の情報を収集する：

- `mcp__miro__context_get` でボード概要を取得
- `mcp__miro__board_list_items` で全アイテム（付箋・カード・テキスト等）を取得（limit: 100）

アイテムが100件を超える場合は `nextCursor` を使って追加取得する。

### Step 3: タスクの洗い出しと構造化

取得したアイテムの内容・色・位置関係から、タスク構造を推定する：

- **付箋の色やサイズの違い** → カテゴリやレベルの区別に活用（例: 大きい付箋=親タスク、小さい付箋=サブタスク）
- **位置の近さ** → 関連するアイテムのグルーピングに活用
- **テキスト内容** → 「〜する」「〜を調べる」等のアクション表現をタスク候補として抽出
- **疑問形（「〜か？」）** → 調査タスクとして変換

構造化の結果を一覧でユーザーに提示し、追加前に確認を取る。内容が多い場合やグルーピングに迷う場合は特に確認を挟む。

### Step 4: TODOリストへの登録

ユーザーの確認後、以下の順で登録する：

1. `mcp__mcp-todo__todo_add` でメインタスクを一括追加（memo欄に補足情報を入れる）
2. 返却されたタスクIDを使い、`mcp__mcp-todo__todo_add_subtask` でサブタスクを追加

### Step 5: 結果報告

追加したタスク・サブタスクの一覧をテーブル形式で表示する。タスク数・サブタスク数の合計も記載する。

## 完了条件

Miroボードの内容がタスク・サブタスクとしてTODOリストに登録され、結果一覧がユーザーに表示された時点で完了。
